# 📖 使用指南

## 快速开始

### 1. 启动开发服务器

```bash
npm run dev
```

访问提示的本地地址(通常是 `http://localhost:5173`)

### 2. 第一次使用

1. 进入**单词库**页面
2. 点击"➕ 添加单词"按钮
3. 添加几个单词,例如:
   - apple - 苹果
   - book - 书
   - happy - 快乐的

### 3. 开始学习

1. 返回**今日学习**页面
2. 查看英文单词,思考中文意思
3. 点击卡片显示答案
4. 根据记忆情况选择:
   - 😊 **记住了** - 单词熟练度提升,延长复习间隔
   - 😞 **忘记了** - 单词熟练度降低,缩短复习间隔

## 核心功能详解

### 📚 单词库

**功能**:
- 查看所有已添加的单词
- 查看每个单词的熟练度、复习次数、下次复习时间
- 添加新单词
- 删除单词

**熟练度等级**:
- 新词 (0级)
- 生疏 (1级)
- 一般 (2级)
- 熟悉 (3级)
- 掌握 (4级)
- 精通 (5级)

### 📖 今日学习

**学习内容**:
- 待复习的单词(已到复习时间)
- 今日新增的单词

**学习流程**:
1. 查看单词
2. 回忆意思
3. 点击显示答案
4. 评估记忆情况
5. 继续下一个

**统计显示**:
- 今日新学单词数
- 今日已复习单词数
- 当前待复习单词数

### 🔄 复习计划

**查看内容**:
- **待复习**: 需要立即复习的单词
- **即将复习**: 未来几天需要复习的单词,按日期分组

**复习时间线**:
基于艾宾浩斯遗忘曲线,系统自动安排:
- 第1次: 5分钟后
- 第2次: 30分钟后
- 第3次: 12小时后
- 第4次: 1天后
- 第5次: 2天后
- 第6次: 4天后
- 第7次: 7天后
- 第8次: 14天后
- 第9次: 30天后

### 📊 学习统计

**总体统计**:
- 总单词数
- 已掌握单词数
- 总复习次数
- 掌握率

**最近7天学习情况**:
- 每天新学单词数
- 每天复习单词数

**单词掌握度分布**:
- 可视化显示各熟练度等级的单词数量

**学习成就**:
- 🌱 初学者 - 学习10个单词
- 🌿 勤奋学习 - 学习50个单词
- 🌳 词汇达人 - 学习100个单词
- ⭐ 精通大师 - 掌握20个单词
- 💪 坚持不懈 - 复习100次
- 🔥 连续打卡 - 连续学习7天

## 💡 学习建议

### 最佳实践

1. **每天坚持学习**
   - 建议每天学习10-20个新单词
   - 完成所有待复习的单词
   - 保持连续学习天数

2. **诚实评估**
   - 如果真的忘记了,选择"忘记了"
   - 不要为了快速完成而选择"记住了"
   - 系统会根据你的真实情况调整复习计划

3. **及时复习**
   - 看到待复习提醒,尽快复习
   - 复习间隔是经过科学计算的
   - 及时复习效果最好

4. **合理安排**
   - 早上学习新单词,记忆效果好
   - 晚上复习当天的单词
   - 利用碎片时间复习

### 记忆技巧

1. **联想记忆**
   - 将单词与熟悉的事物联系
   - 创造有趣的记忆场景
   - 使用谐音、词根等方法

2. **语境记忆**
   - 将单词放在句子中理解
   - 想象使用场景
   - 制作例句(可以添加到中文释义中)

3. **重复复习**
   - 按照系统提示的时间复习
   - 不要跳过复习环节
   - 重复是记忆之母

## 🔧 常见问题

### Q: 数据会丢失吗?

A: 数据保存在浏览器的 LocalStorage 中,只要不清除浏览器数据,就不会丢失。但建议:
- 不要清除浏览器缓存和数据
- 未来会添加导出功能,届时可以备份数据

### Q: 可以在手机上使用吗?

A: 可以!项目支持响应式设计,可以在手机浏览器中访问。在电脑上运行开发服务器后,手机连接同一WiFi,访问电脑的IP地址和端口即可。

### Q: 如何批量添加单词?

A: 目前需要手动逐个添加。批量导入功能在未来计划中,届时可以导入Excel/CSV文件。

### Q: 可以修改单词吗?

A: 当前版本不支持修改,只能删除后重新添加。编辑功能会在后续版本中添加。

### Q: 复习时间可以自定义吗?

A: 当前使用固定的艾宾浩斯遗忘曲线间隔,暂不支持自定义。这个间隔是经过科学验证的最佳复习时间。

## 📱 部署到生产环境

### 构建项目

```bash
npm run build
```

构建完成后,`dist` 目录包含所有静态文件。

### 部署方式

1. **GitHub Pages**
   - 将 `dist` 目录推送到 `gh-pages` 分支
   - 在仓库设置中启用 GitHub Pages

2. **Vercel**
   - 连接 GitHub 仓库
   - 自动检测 Vite 项目
   - 一键部署

3. **Netlify**
   - 拖拽 `dist` 目录到 Netlify
   - 或连接 Git 仓库自动部署

4. **服务器**
   - 上传 `dist` 目录到服务器
   - 配置 Nginx/Apache 指向该目录
